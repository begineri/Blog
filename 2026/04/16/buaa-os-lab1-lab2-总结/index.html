lab2 重点

其实lab2就分了三个部分

1. 物理内存

只需理解以下几点：

  1. 操作系统启动时：会探明物理内存有多大，然后建一个超级大数组（pages），把物理内存按 4KB 切成一块块，每一块分配一个 struct Page
  2. 物理内存分页：4kb每页
  3. 空闲内存池：构建一个 free list pool，核心时不用每次申请内存都到大费周章去从内存中malloc一个，而是直接从我们构建好的空闲链表头取出一个空间即可，归还时同样直接把这个块插回到链表头
  4. 双向链表：实现了O(1)的增删改查复杂度，方便快捷。理解重点是那个二级指针：始终指向的是指向你的那个箭头（还是便于直接修改前一个节点指向的内容）。

2. 虚拟内存管理

  1. 虚拟地址：PDX (高10位) + PTX (中10位) + Offset (低12位)。
  2. 虚拟地址 <–> 物理地址 ：通过CPU漫长的访存，或TLB，这张图很好地展现了这几个地址间的转换关系：
  3. pgdir_walk / page_insert / 自映射：详情见下方难点分析

3. TLB 清除与重填

  1. TLB目的：加速CPU的访存
  2. TLB 重填 (Refill)：do_tlb_refill，写tlb
  3. TLB 清除 (Invalidate)：当页表项修改，tlb_invalidate → tlb_out
  4. 始终保证 CPU 缓存（即tlb）与内存的绝对一致性

难点分析：

上面的内容听起来很简单，但实际上只靠阅读ppt + 指导书 + 看网课 + 甚至问AI，我在学校过程中仍然遇到了很大的理解困难。归根到底，我觉得晦涩难懂的翻译和ppt或指导书上完全让人无法理解的遣词造句，是主要原因：毫不直观，没有“为什么需要这个”，没有“为什么是这样构造的”，只会一上来给你全部的概念，让人自己尝试练习阅读理解。

看个指导书上的例子，上来就是一个长难句起手：

“ int pgdir_walk(Pde *pgdir, u_long va, int create, Pte **ppte)，该函数将一级页表基地址 pgdir 对应的两级页表结构中 va 虚拟地址所在的二级页表项的指针存储在 ppte 指向的空间上。 ”

一句话里涉及到了无数个概念：

  1. 一级页表：Page Directory（页目录，不如直接叫页表-表），指存储所有页表的那张表
  2. 一级页表基地址：：即参数 pgdir，page dir这张大表的首地址
  3. 两级页表结构：指的是一级页表+二级页表的这个结构（废话）
  4. 虚拟地址 va：重点，我们传入的数据，通过它来进行查找
  5. 二级页表：Page Table，页表，真正存所有页表项的地方
  6. 二级页表项的指针：指向页表的一个指针
  7. ppte 指向的空间上：ppte是什么？是一个指针，指向上面说的那个找到的二级页表，返回它作为结果

翻译一下这句话，就是：pgdir_walk 这个函数的功能是，给定一个虚拟地址 va，cpu 顺着 pgdir 往下找，找到记录 va 对应的物理页的那张页表，然后返回页表的指针，便于我们对其进行修改。

同样的，对于page_insert：

“int page_insert(Pde *pgdir, u_int asid, struct Page *pp, u_long va, u_int perm)，作用是将一级页表基地址 pgdir 对应的两级页表结构中虚拟地址 va 映射到页控制块 pp 对应的物理页面，并将页表项权限为设置为 perm。”

关键概念：

  1. 页控制块 pp： 一个结构体指针 struct Page *，直接对应着一个物理页
  2. 页表项权限：规定pp对应这块内存是“只读”、“可读写”还是“仅内核可见”。
  3. 这里用到page2pa：把 Page 指针转成物理地址

翻译过来就是：给定va，page table，找到对应那个页表，然后填写页表项（使用va、pp、perm）。

对于 page_lookup:

“struct Page * page_lookup(Pde *pgdir, u_long va, Pte **ppte)，作用是返回一级页表基地址 pgdir 对应的两级页表结构中虚拟地址 va 映射的物理页面的页控制块，同时将 ppte 指向的空间设为对应的二级页表项地址。”

拆解：

  1. 两级页表结构中虚拟地址 va 映射的物理页面的页控制块：va -> struct Page*，作为返回值，返回va对应的物理块
  2. 将 ppte 指向的空间设为对应的二级页表项地址: *ppte = pgdir_walk(va) 找到的那个页表项地址

总结这个函数的功能，就是通过va，返回目前所有页表项里查找到其对应的物理块，同时把记录这个数据的页表项返回，便于修改。

就像数学，很多人觉得那些概念看上去很难，实际上只是他们对这个数学概念中所蕴涵的组成概念陌生，因而导致了对整体概念的无法理解。因此正如上面的过程，把一个长长的概念拆开，会发现里面包含了无数细小的概念，必须理解那些概念以后，才能对整体有了理解。

在这次lab接触自映射这个概念时，我对着ppt苦思冥想，一头雾水，甚至ai都没给我说明白，一会是物理地址，一会又要转虚拟地址，为什么？是什么？完全没有解答，只有ppt上密密麻麻的字和不知所云的图片。在我拼命尝试理解的那个下午，我以为这是神秘而高深的概念。

当我终于阅读了C语言的代码，我发现这不过是几个指针访问与赋值的技巧：
改一下指导书里出现的这张图：

四个32位的格式不同的表头指来指去，我最初看到的时候还以为这里访问了多次地址，其实，这里就是一个获取指针（地址）与指针解引用（数据）的过程，而这张图完全没有区分出来。

总之过程如下（va -> 页表项，即pgdir_walk的过程）：

  1. (pgdir | va[31:22] | 00) -> PDX* 类型，假设命名为a
  2. 对a解引用 (*a) -> 得到PTB的基地址（这是物理地址，为了cpu能访问到，需要转为虚拟地址）
  3. PTBbase | va[21:12] | 00 -> PTB* 类型，假设命名为b
  4. b 就是我们最终要找的这个页表项，返回它，可以用于后续对页表项的添加或修改。

这是这张图所表达的，而所谓“自映射”，就是为CPU提供一个便利：CPU只会使用这套流程去找数据，但这样只能找到页表项。CPU为了直接能修改页表甚至页表表（页目录），设置了一个巧妙的方式：即把页目录本身就插入为众多页表中，这样访问的时候，就可以使用：pgdir[a] -> pgdir, padir[b] -> 某个页表（而不是页表项）

通过同样的流程，CPU可以获得并修改整个页表。

这里的为什么，怎么做，指导书以及ppt均没有点明，只能靠读者去猜了。
